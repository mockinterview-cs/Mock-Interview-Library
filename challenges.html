Recursion

Find prime numbers in a string
Last updated: Nov 15th 2022
Given a numeric string. Return a boolean value indicating whether there's any prime number in the string.

For example:
console.log(findPrime('849623')); // -> true (2, 3, 23, 9623)
console.log(findPrime('468688646')); // -> false
console.log(findPrime('2')); // -> true (2)
Check occurrences of a number
Last updated: Nov 6th 2022
Given an arbitrarily nested array of arrays, a number N and an integer K. Return 'YES' if the number N appeared exactly K times in the given array, otherwise, return 'NO'.

For example:
console.log(doesNumAppearKTimes([1,2,[3,4,[5],6],3,7],3,3)); // 'NO'
console.log(doesNumAppearKTimes([[0,-8,1,[-2,2]],[11,[4],6],2,2],2,3)); // 'YES'
console.log(doesNumAppearKTimes([10,21,4,9,3,7,4,1],4,1)); // 'NO'
console.log(doesNumAppearKTimes([-8,[-1,-2,-7],[-1],-3,-8],-1,3)); // 'NO'
console.log(doesNumAppearKTimes([2,[3,[4,[5,[6,[],6],5],4],3],2],6,1)); // 'NO'
console.log(doesNumAppearKTimes([2,9,[11,11,11],0],0,1)); // 'YES'
console.log(doesNumAppearKTimes([[[[[[[[[[[[7]]]]]]]]]]]],7,0)); // 'NO'
console.log(doesNumAppearKTimes([[91],[0],[22],[-18],[107],[],[21]],21,1)); // 'YES'
Validate basic JS expressions
Last updated: Nov 6th 2022
Given a string that can contain only: numeric characters, '+' signs, '-' signs and spaces. Return a boolean value indicating whether the input string is a valid expression in JS.

For example:
console.log(isValidExp('-21+80+917-1+-2')); // true
console.log(isValidExp('+11+0+29-11+--3-')); // false
console.log(isValidExp('29+18+9-77++-23')); // false
console.log(isValidExp('-5+9+2-1-+3')); // true
console.log(isValidExp('+-2+8+9-1- - + - + +3+')); // false
console.log(isValidExp('+37-1+19-1- - + - + +2-16 +-7-2 + 0')); // true

BrowserHistory
Declare a function 'BrowserHistory', which takes in no argument and returns an instance of a 'BrowserHistory' object when invoked with the 'new' keyword.
BrowserHistory objects should mimic the basic function of a history tracker - i.e. they will keep track of the order of searched items.
All BrowserHistory objects should contain their own 'size' and 'index' properties. The ‘size’ keeps track of the number of items in the object. The 'index' works as a 'pointer' to reference the current item. When a BrowserHistory is first created, it should have a size and an index of 0.
- Task 1: Declare a function 'search', which is accessible to ALL instances of 'BrowserHistory'. 'search' should take in a single argument, and when invoked on a BrowserHistory instance, should add the passed-in argument to the object as a key, and the corresponding value is another object with only one property called 'order'. The 'order' is an array of numbers to keep track of the order of invocation on the item. When a new item is added, it should be added to the highest order (the highest order is the latest added item and the lowest order is the oldest added item), and if an item has already existed in the object, push the new order to the order list of that item. Update the 'index' to the latest added order. And update the size properly as well.
- Task 2: Declare a function 'goBack', which is accessible to ALL instances of 'BrowserHistory'. 'goBack' should take in no argument, and when invoked on a BrowserHistory instance, should return the previous searched item compared to the current one. Return an empty string if not found! During the execution, adjust the 'index' properly to keep track of the history (i.e. decrement the 'index' by 1).
- Task 3: Declare a function 'goForward', which is accessible to ALL instances of 'BrowserHistory'. 'goForward' should take in no argument, and when invoked on a BrowserHistory instance, should return the next searched item compared to the current one. Return an empty string if not found! During the execution, adjust the 'index' properly to keep track of the history (i.e. increment the 'index' by 1)
- Task 4: Declare a function 'clean', which is accessible to ALL instances of 'BrowserHistory'. 'clean' should take in a positive integer K as a single argument, and when invoked on a BrowserHistory instance, should remove the first K oldest searches among all the searched items (i.e. the first K smallest orders). If the order list is empty, remove the corresponding item from the object and update the size properly.
Domino
Last updated: Nov 17th 2022
Declare a function 'Domino', which takes in a single number as an argument (called 'nextIndex') and returns an instance of a 'Domino' object when invoked with the 'new' keyword. Domino objects should function similarly to Arrays -i.e. they will store values at numbered indexes, starting from 0. However, numbered indexes in Domino objects don't need to be consecutive!
All Domino objects should contain their own 'chainLength' property, which keeps track of the number of items they contain. When a Domino is first created, it should initialize the value at 0-th index to the 'nextIndex', and also set the value at the nextIndex-th index to null, and it should have the chainLength of 1.
- Task 1: Declare a function 'add', which is accessible to ALL instances of 'Domino'. 'add' should take in a single number as an argument, and when invoked on a Domino instance, should set the value of the key that has a null value to the passed-in number, and set the value at the number-th index to null. However, if the added number has already existed as a key in the object (and not holding the null value), remove the property associated with that number from the object. Update the Domino's chainLength property, and returns the new chainLength.
- Task 2: Declare a function 'remove', which is accessible to ALL instances of 'Domino'. 'remove' should take in a number as a single argument, and when invoked on a Domino instance, should update the value of the key containing the passed-in number to null and remove the chain starting from that number. Update the Domino's chain length property and return the new chain length. However, if the passed-in number is not found as a key or there's no link TO that number, return "Oops! Something went wrong --Can't remove the *number*.".
*Important note: After removing an element, there is a possibility that in the current object there will be more than one key with a null value. In this case, keep only the largest key.
- Task 3: Declare a function 'count', which is accessible to ALL instances of 'Domino'. 'count' should take in no arguments, and when invoked on a Domino instance, should return the number of connected components.
- Task 4: Declare a function 'checkPath' which is accessible to ALL instances of 'Domino'. 'checkPath' should take in two numbers (called 'source' and 'target') as arguments, and when invoked on a Domino instance, should return the number of steps to go from 'source' to 'target' if there's a path FROM source TO target (one-way path), otherwise return "NO PATH!".
OrderedList
Last updated: Nov 14th 2022
Declare a function 'OrderedList', which takes in no arguments and returns an instance of an 'OrderedList' object when invoked with the 'new' keyword. OrderedList objects should function similarly to arrays - i.e. they will store values at numbered indexes, starting at 0. However, all elements are in sorted order.
All OrderedList objects should contain their own 'length' property, which keeps track of the number of items they contain. When an OrderedList is first created, it should have a length of 0.
- Task 1: Declare a function, 'add', which is accessible to ALL instances of 'OrderedList'. 'add' should take in a number as a single argument, and when invoked on an OrderedList instance, should add the number to that OrderedList in descending order, update the OrderedList's length property appropriately, and return the length of the OrderedList.
- Task 2: Declare a function, 'reverse', which is accessible to ALL instances of 'OrderedList'. 'reverse' should take in no arguments, and when invoked on an OrderedList instance, should reverse the order of all elements in that OrderedList (except the 'length' property).
- Task 3: Declare a function, 'getMax', which is accessible to ALL instances of 'OrderedList'. 'getMax' should take in no arguments, and when invoked on an OrderedList instance, should return the maximum number in the instance.
LimitedContainer
Last updated: Nov 12th 2022
Declare a function 'LimitedContainer', which takes in a single argument (called 'threshold') and returns an instance of a 'LimitedContainer' object when invoked with the 'new' keyword.
LimitedContainer objects should function similarly to objects - i.e., they will store elements as key-value pairs, and the value is the number of times the key appeared in the object.
All LimitedContainer objects should contain their own size and threshold properties. The size keeps track of the number of items a LimitedContainer instance contains and the threshold is the upper-limit of a value in the object. When a LimitedContainer is first created, it should have a size of 0.
- Task 1: Declare a function, 'add', which is accessible to ALL instances of 'LimitedContainer'. 'add' should take in a single argument as a string, and when invoked on a LimitedContainer instance, should update the number of times that value appears as a key to that LimitedContainer. However, if any value is equal to the threshold, remove the corresponding property from the instance. Update the LimitedContainer's size property appropriately, and return the new size of the LimitedContainer.
- Task 2: Declare a function 'group', which is accessible to ALL instances of 'LimitedContainer'. 'group' should take in no arguments, and when invoked on a LimitedContainer instance, should group all keys with the same value as a property with the key is the value and the value is an array containing all the keys. The LimitedContainer's threshold remains unchanged but the size property needs to be appropriately updated.
- Task 3: Declare a function 'check', which is accessible to ALL instances of 'LimitedContainer'. 'check' should take in a single argument as a string, and when invoked on a LimitedContainer instance, should return the number of times that argument appeared in the instance.
- Task 4: Declare a function 'ungroup', which is accessible to ALL instances of 'LimitedContainer'. 'ungroup' should take in no arguments, and when invoked on a LimitedContainer instance, should ungroup all properties with the keys are the original properties as strings and the values are the number of time a particular key appeared in the instance as integers. The LimitedContainer's threshold remains unchanged but the size property needs to be appropriately updated.
* Bonus: Make the function 'add' works properly when the instance is grouped!
DuplicatedContainer

DuplicatedContainer objects should function similarly to objects - i.e., they will store elements as key-value pairs, and the value is the number of times the key appeared in the object.

Storage class
Last updated: Nov 9th 2022
Declare a class 'Storage' with 2 properties: size and capacity. The 'size' property keeps track of the number of current elements in the storage. The 'capacity' property is the maximum number of elements that storage can safely store. 'Storage' classes should function similarly to an Array - i.e. they will store values at numbered indexes, starting at 0.
- Task 1: Add a constructor to this class. The constructor takes in a single argument for the capacity property and initializes the size and the capacity. When a Storage is first created, it should have a size of 0.
- Task 2: Declare a function 'add', which is accessible to ALL instances of 'Storage'. 'add' should take in a single argument, and when invoked on a Storage instance, should add the argument to the first possible position of that Storage and return the size of the instance. However, if the 'capacity' is exceeded, return "Maximum size exceeded!".
- Task 3: Declare a function 'remove', which is accessible to ALL instances of 'Storage'. 'remove' should take in a single argument which is the value that needs to be removed, and when invoked on a Storage instance, should remove all the properties with the corresponding value from the instance and return the size of that instance. If the value doesn’t exist, return "No values found!".
